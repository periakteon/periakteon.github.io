Fonksiyonlar ile Yeniden Kullanılabilir (Reusable) JavaScript Yazmak

JavaScript'te, kodumuzu fonksiyonlar adı verilen yeniden kullanılabilir (reusable) parçalara ayırabiliriz.

function functionName() {
console.log("Hello World");
}

Yazdığım fonksiyonları şu şekilde çağırabiliriz (call) veya çalıştırabiliriz (invoke):
functionName();

Fonksiyon her çağrıldığında geliştirici konsoluna Hello World mesajı yazdırılır. Küme parantezleri arasındaki tüm kodlar, fonksiyon her çağrıldığında yürütülür.

function reusableFunction() {
console.log("Hi World");
}
reusableFunction();

>>> Hi World

Argümanlar ile Değerleri Fonksiyonlara Geçirmek/İletmek (Passing)

Parametreler, çağrıldıklarında bir fonksiyona girilecek değerler (values) için yer tutucu görevi gören değişkenlerdir (variables). Bir fonksiyon tanımlandığında, genellikle bir veya daha fazla parametre ile birlikte tanımlanır. Çağrıldığında bir fonksiyona girilen (veya "geçirilen" / "passed") gerçek değerler argüman olarak bilinir.

Aşağıdaki örnekte iki parametreye, yani param1 ve param2 adlı iki parametreye sahip bir fonksiyon örneği vardır:

function funcWithTwoParam(param1, param2) {
console.log(param1, param2);
}

Bu fonksiyonu şöyle çağırdığımızı düşünelim:

funcWithTwoParam("Hello", "World");

Bu fonksiyondaki parametrelerin yerine iki string geçirdik, yani parametrelerin yerine Hello ve World koymuş olduk.

Fonksiyon içinde, param1 adındaki parametre esasen Hello değerindeki bir stringe ve param2 adındaki parametre de World değerine sahip bir stringe eşit olacaktır. funcWithTwoParam'ı farklı bağımsız değişkenlerle tekrar çağırabileceğinizi ve parametrelerin yeni bağımsız değişkenlerin değerini alacağını unutmayın.

function toplamaFonksiyonu(a, b) {
console.log(a + b);
}
toplamaFonksiyonu(5, 7);

>>> 12

return ile Fonksiyondan Değer Döndürmek

Değerleri argümana sahip bir fonksiyona aktarabiliriz . Bir fonksiyondan geri bir değer göndermek için return ifadesi kullanabiliriz.

function plusThree(number) {
return num + 3;
}

const answer = plusThree(5);

Yukarıdaki answer değişkeni artık 8 değerine sahiptir.

plusThree adındaki fonksiyon esas itibariyle number için bir argüman alır ve bize number + 3'e eş bir değer döndürür.

function timesFive(num) {
return num * 5;
}

const answer = timesFive(6);

console.log(answer);

>>> 30

Global Scope ve Fonksiyonlar

JavaScript'te kapsam / kapsama alanı (scope), değişkenlerin görünürlüğünü ifade eder. Bir fonksiyon bloğunun (yani, {}) dışında tanımlanan değişkenler Global kapsama sahiptir. Bu, bu değişkenlerin JavaScript kodunun her yerinde görülebilecekleri (ve dolayısıyla da erişilebilecekleri) anlamına gelir.

let veya const anahtar sözcükleri olmadan tanımlanan değişkenler otomatik olarak global kapsamda oluşturulur. Bu, kodumuzun başka bir yerinde veya bir fonksiyonu tekrar çalıştırırken istenmeyen sonuçlara neden olabilir. Değişkenlerinizi her zaman let veya const ile tanımlamalısınız.

//let ile global değişken tanımlıyoruz
let myGlobal = 10;

function fun1() {
//keyword kullanmadan tanımladığımız için global:
oopsGlobal = 5;
}

function fun2() {
let output = "";
if (typeof myGlobal != "undefined") {
output += "myGlobal: " + myGlobal;
}
if (typeof oopsGlobal != "undefined") {
output += " oopsGlobal: " + oopsGlobal;
}
console.log(output);
}

fun2();

>>> myGlobal: 10 oopsGlobal: 5

Ayrıca bkz.: https://www.w3schools.com/js/js_scope.asp

Local Scope ve Fonksiyonlar

Bir fonksiyon içinde tanımlanan değişkenlerin yanı sıra fonksiyon parametreleri de local (local) kapsama sahiptir. Bu, yalnızca o fonksiyon içinde görünür oldukları anlamına gelir.

Aşağıda localScope adındaki bir değişkene sahip bir fonksiyon örneği vardır:

function myTest() {
const localScope = "Hey, ben localScope!";
console.log(localScope);
}

myTest();
console.log(localScope);

>>> Hey, ben localScope!
>>> ReferenceError: localScope is not defined

myTest() adındaki fonksiyon çağrıldığında, konsolda Hey, ben localScope! olarak bir string döndürür.

Hemen ardından gelen console.log(localScope) satırı ise bize bir hata verir çünkü localScope adındaki değişken aslında myTest adındaki fonksiyon içerisinde tanımlanmıştır, yani local bir değişkendir ve dolayısıyla fonksiyon dışarısından erişilemez.

Fonksiyonlarda Global vs. Local Scope

Aynı ada sahip hem local hem de global değişkenlere sahip olmak mümkündür. Bu durumda, local değişken global değişkene göre önceliklidir.

const someVar = "Hat";
function myFun() {
const someVar = "Head";
return someVar;
}

myFun değişkeni çağrıldığında Head şeklinde bir string döndürecektir çünkü yukarıda da belirtildiği üzere local değişkenin global değişkene önceliği vardır.

const outerWear = "T-Shirt";

function myOutfit() {
const outerWear = "sweater"
console.log(outerWear);
}

myOutfit();

>>> sweater
