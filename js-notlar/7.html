Bir Fonskiyondan Dönen Undefined Value'yu Anlamak

Bir fonksiyon, return ifadesini içerebilir ancak içermesi zorunlu değildir. Fonksiyonun bir return ifadesini içermemesi durumunda ve fonksiyon çağrıldığından, fonksiyon içteki kodu işler ancak döndürülen (returned) değer undefined'dır.

let sum = 0;

function addSum(num) {
sum = sum + num;
}

addSum(3);

Buradaki addSum esas itibariyle return ifadesi içermeyen bir fonksiyondur. Bu fonksiyon, global bir scope'a sahip olan sum değişkeninin değerini değiştirecek fakat returned value ise undefined olacaktır (çünkü fonksiyon içerisinde return ifadesi kullanılmamıştır).

Returned Value'yu Atamak

Değişken Tanımlama dersindeki tartışmamızdan hatırlayacağımız gibi, eşittir (=)işaretinin sağındaki her şey değer atanmadan önce işlem görür. Bu, bir fonksiyonun dönüş değerini alıp bir değişkene atayabileceğimiz anlamına gelir.

İki sayıyı toplayan bir toplam fonksiyonuu tanımladığımızı varsayalım.

ourSum = sum(5,12);

sum adındaki fonksiyona iki argüman, yani 5 ve 12 verdiğimizde alacağımız return value esasen 17'dir. Bu returned value de ourSum adındaki değişkenimize atanır (= operatörünün sağındaki işlemlerin daha önce işlem ördüğünü yukarıda belirttik).

let processed = 0;

function processArg(num) {
return (num + 3) / 5;
}

processed = processArg(7);

console.log(processed);
