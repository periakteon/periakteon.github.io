Stringlerde Literal Alıntılardan Kaçmak

Bir string türünde değişken tanımlamak için çift tırnak "..." veya tek tırnak '...' kullanırız. Ancak tanımladığımız string türündeki değişkenin içerisinde alıntılar varrsa veya yine tırnak kullanmamız gerekiyorsa ne yapacağız?

Bunun için alıntının veya tırnak işaretiyle başlayan ifadenin önüne backslash, yani (\) ekleriz. Bu, JavaScript'e ilgili alıntının stringin sonu olmadığını, bunun yerine stringin içinde görünmesi gerektiğini bildirir.

const sampleStr = "Alan said, \"I'm coming\".";

console.log(sampleStr);

>>> Alan said, "I'm coming".

------------------------------------------

const myStr = "I am a \"double quoted\" string inside \"double quotes\".";

console.log(myStr);

>>> I am a "double quoted" string inside "double quotes".

Stringleri Tek Tırnakla Alıntılama

JavaScript'teki string değerleri, aynı türdeki alıntı işaretiyle başlayıp bittiği sürece tek ( '...' ) veya çift tırnak ( "..." ) ile yazılabilir. < a > etiketiyle başlayan string değerlerinde bu kullanılabilir.

const myStr = '< a href="http://www.example.com" target="_blank"> Link < / a>';

console.log(myStr);

>>> < a href="http://www.example.com" target="_blank"> Link < / a >

Stringlerde Kaçış Dizileri

Kaçış dizilerini kullanmanın iki nedeni vardır:

I) Meselâ yeni satıra geçmek gibi durumlarda veya başka türlü yazılması mümkün olunmayan durumlarda,
II) Bir stringte çoklu alıntıya imkân sağlamak için.

Kaçış dizelerine aşağıdaki tablodan ulaşabilirsiniz:

Code Output
 \' single quote
 \" double quote
\\ backslash
\n new line
\t tab
\r carriage return
\b word boundary
\f form feed


Bir örnek verelim:

const myStr = "FirstLine\n\t\\SecondLine\nThirdLine";

console.log(myStr);

>>>
FirstLine
 \SecondLine
ThirdLine

Stringleri  +  Operatörüyle Birleştirme

Stringleri birleştirmek için...

const myStr = "as";

console.log(myStr);

>>> as
